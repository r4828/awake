# Awake 2.0.7

Awake now recovers closed-lid sleep settings after a daemon crash instead of leaving `pmset disablesleep 1` behind.

## Fixed

- launchd now owns the foreground daemon and restarts it after crashes or `kill -9`
- clean `awake stop` exits without triggering a restart
- startup restores Awake-owned power settings when no valid agent, timer, rule, command, or manual session remains
- recovery checks the real `disablesleep` value and preserves user-owned power settings
- failed restoration keeps enough state to retry on the next reconciliation
- timers publish their PID and lease atomically, including timer replacement
- stale, malformed, and orphaned leases no longer keep the Mac awake
- `awake install` now bundles `private-apis.h` and fails when the native app build fails

## Onboarding

- removed the optional menu bar control permission step
- kept the menu bar icon and the onboarding hint that shows where to find Awake

## Install or update

```bash
npx --yes awake-agent@latest install
```
