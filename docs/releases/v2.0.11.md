# Awake 2.0.11

Awake installation now succeeds only when the newly installed menu-bar app is actually running.

- installation restarts an older Awake UI process and verifies the new process stays alive
- `awake ui` uses the same verified launch path instead of returning success after a failed open
- release packaging cannot stop or replace the currently running Awake app
- the installed workflow test now covers a real five-minute timer, detached-shell survival, manual modes, command leases, automatic agent protection, runtime health, and timer cancellation
- agent detection tests cover Claude hooks, Codex hooks, Codex, Aider, Copilot, Amp, OpenCode, stale heartbeats, and Aider’s Python fallback
- test hook and runtime state are isolated from live Claude, Codex, and launchd sessions
